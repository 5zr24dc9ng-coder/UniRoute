# UX

> 更新頻度：低
> ChatGPT / Lovable / Claude Code向け。新しい画面・機能を作る前にここを確認する。

## ユーザー導線

実装済みのナビゲーション（`NAV`定義、サイドバー上から順）：

```
シミュレーション（sim）
  ↓
比較（matrix）
  ↓
タスク管理（tasks）
  ↓
ビザ費用（visa）
```

これに加えて、ログイン不要で到達する2つの独立導線がある。

- 家族向け共有レポート（`?share=1&...`）：ユーザーが「共有」操作をした結果生成されるURLをきっかけに、家族・保護者がログインなしで見る導線
- 規約ページ（`?legal=terms|privacy|tokushoho`）：フッターリンクやClerkの同意リンクから到達

アフィリエイト（Wise送金比較）は「シミュレーション」画面内の送金コスト比較セクションに埋め込まれており、独立したページ・導線ではなく本流の中に自然に置く方針。今後、航空券・保険の比較を追加する場合も同様に「シミュレーション結果を見た流れの中で提示する」形を踏襲する（別ページに切り出して回遊させる設計にはしない）。

## UX原則

**クリック数**
主要な数値（総額・ビザ資金証明額）はシミュレーション画面を開いた瞬間に見える状態にする。追加の入力（都市ティア・留学期間・スカラシップ等）は任意項目として後から調整できる形にし、初期表示のために必須の操作を増やさない。

**CTA**
プライマリCTA（購入・アップグレード）は`#2f63e6`の単色ボタンに統一し、キャンセル・閉じるなどの副次操作と視覚的に混同しない。未ログイン時に「プレミアムを購入」を押した場合はいきなり決済に進めず、先にログイン導線へ誘導する（実装済み：`d7c6e37`のコミットで意図的にこの順序に変更済み）。

**入力項目**
入力はスライダー（range input）を優先し、テキスト入力は最小限にする（数値の手入力ミスや全角半角の揺れを避けるため）。ただし「毎月貯蓄に回せる金額」のような個別性の高い金額はテキスト入力を許可する。

**エラー表示**
Webhook・API層のエラーは実際のエラー内容をログに残す（詳細を隠さない）。ユーザー向けの画面上のエラー表示は今のところ最小限（サービス全体が「エラーが起きたらまず動かない」ことに気づける状態を優先しており、握りつぶして正常に見せることはしない）。

## 禁止事項

- **ダークパターン・情報の意図的な隠蔽の禁止**：法定表示（特商法・利用規約・プライバシーポリシー・アフィリエイトのPR表記）や同意まわりで、視認性を下げる・分かりにくくする設計は一切行わない。これはKaiの明確な方針であり、交渉の余地がない前提として扱う
- **プレミアム判定のlocalStorage永続化禁止**：`?admin=true`によるプレミアム解放は、セッション中のみ有効とし、絶対にlocalStorageへ保存しない。過去に永続化していたことで「誰でも推測できるURLを一度踏んだ端末で永久にプレミアムが解放される」バグが実際にあったため、二度と同じ設計にしない
- **なりすまし・ステルスマーケティング的な導線の禁止**：知恵袋・SNS等への回答やレビュー投稿は必ず運営者本人であることを開示した上で行う（景品表示法のステマ規制、2023年10月施行）。「困っているふりをしてURLを紛れ込ませる」ような投稿は一切行わない
- **絵文字の使用は最小限**：新規UI文言・新規コードコメントには使わない
